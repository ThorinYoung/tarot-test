## BGM 接入(自动槽)

- 把生成好的音乐命名为 **`bgm.mp3`** 放进本目录(与 index.html 同级),刷新即自动启用(WebAudio 无缝循环 + 淡入,♪ 按钮控制);没有该文件时回退为内置生成式钟音。
- 音量在 `game.js` 顶部 `BGM_VOL`(默认 0.45)调整。
- 规格建议:**纯器乐、90-120 秒、动态平稳(无大起大落的 intro/outro)、首尾不留长静音**;128-192kbps mp3(1-3MB),GitHub Pages 无压力。
- ⚠️ **商用授权**:Suno/Udio 等需付费订阅档(各 ~$10/月起)生成的曲目才有商用权,**正式上线前务必用付费账号重新生成并留存订阅凭证**;demo 内部测试不受影响。
- 正式版规划:BGM 走远程 bundle/CDN 加载,不占微信小游戏主包(docs/01 架构)。

## 本地运行

- 方式一:直接双击 `index.html`(file:// 可玩)。
- 方式二:`python3 -m http.server 4173 --directory demo` 后访问 `http://localhost:4173`。
- 竖屏体验最佳;桌面浏览器会自动居中为手机框。

## 部署到 GitHub Pages

> ⚠️ **安全提醒(重要):不要把整个 tarot 项目仓库公开**——`docs/` 与规划书属于商业机密。
> 正确做法:**单独新建一个仓库,只放本目录的 4 个文件**。

1. 新建 GitHub 仓库(如 `starlight-verdict-demo`,可设 Public)。
2. 把 `index.html`、`style.css`、`game.js`(README 可选)放到仓库**根目录**,push。
3. 仓库 Settings → Pages → Source 选 `main` 分支 `/ (root)` → Save。
4. 约 1 分钟后访问 `https://<你的用户名>.github.io/starlight-verdict-demo/`。
5. 内测分发:直接发链接;手机浏览器/微信内打开均可(微信内置浏览器已验证兼容的 API 范围:Canvas/WebAudio/CSS,无特殊权限)。
